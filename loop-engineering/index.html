04 · 核心原则

🔑 验收分离

写代码的和验收的必须分开。Claude Code 用大模型写、Haiku 验——两个模型各司其职。

🧱 Loop 四件套

触发器 + 技能 + 状态文件 + 门禁。缺一不可。

⚡ 最小可行顺序

① 手动跑通 → ② 写成 skill → ③ 包进 loop → ④ 上定时。跳步是死因。

💰 成本显性化

每个循环的账单必须可见。不知花多少就不是管理，是赌博。

🏢 Boris Cherny (Anthropic) 的工作流

自述去年 11 月卸载了 IDE，一个月没打开过。手下几百个小 Agent 同时跑——扫 GitHub issue、读 Slack 用户反馈、监控 CI 失败。

每个 Agent 在隔离分支里干活：一个写代码，另一个跑测试验收。搞不定的才进他收件箱。

自 Opus 4.5 以来，他的所有代码都是 Claude Code 写的，大部分直接在手机上完成。

📊 Uber 和微软的真实账单

Uber 给 ~5000 名工程师铺开 Claude Code 后，四个月烧穿了 2026 年全年 AI 预算。人均月支出 $150-250，重度用户 $500-2000。

微软在财年末要求数千名工程师从 Claude Code 迁回 GitHub Copilot，官方理由"工具链统一"，被普遍解读为真实动因是成本。
