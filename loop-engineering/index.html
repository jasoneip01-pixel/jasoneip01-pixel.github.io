🌀 Loop Engineering

AI 编程的第四次范式跃迁——从"写提示词"到"设计让提示词自己转起来的系统"

"Nobody writes prompts anymore. The new job is to write and handle loops."

1 什么是 Loop？

一个定义

Loop（循环）就是一段预先写好的逻辑：它自己找出有什么活要干，把一块丢给 AI，检查交回来的东西，不合格就带着报错再丢一次，直到通过或撞到上限才停。你不再是那个写提示词的人，而是那个写"写提示词的东西"的人。

🧑 旧范式：Prompt Engineering

人：写一句提示词 → AI：输出一段代码 → 人：看结果不满意 → 人：再写一句提示词 → ...

每一步都要人盯着、出声。人是"操作员"。

🤖 新范式：Loop Engineering

人：写一个目标+验收标准 → Loop 自己反复提示 AI → AI 自己跑测试 → 不过就改 → 通过或卡住才叫人。

人是"规则设计者"。

2 四次范式跃迁

2023 – Prompt Engineering
怎么跟模型说话
怎么写好单次提示词，诱导模型输出。"会不会写 prompt = 会不会用 AI"。
2024–2025 – Context Engineering
给模型看什么
关注点从"怎么问"变成"提供什么背景"。信息组织能力比写 prompt 更重要。
2026.02 – Harness Engineering
模型在什么环境运行
给 AI 搭能干活、能约束、能调取真实世界资源的脚手架。工具集成、权限边界。
2026.06 – Loop Engineering
让系统自己转起来
从单次执行能力升级为闭环系统的自运行能力。控制粒度上移到"行为级"。

3 核心原则

写代码的 ≠ 验代码的

整个 loop 设计里最重要的原则：写代码的和验收的必须分开。Claude Code 把这条原则直接写进了产品架构——写代码的是大模型，验收的是另一个独立的小模型 Haiku。两个模型各司其职。

为什么这很重要？因为模型给自己写的代码打分时，手太松了。所有"看起来不错"的代码，在独立验收器面前都能挑出一堆毛病。

⏰ 触发器

定时跑、事件触发都行。Claude Code 里用 /loop，Codex 里用 Automations 面板。

🔧 技能

把项目上下文写进 STATE.md 文件，让每次运行不用重新解释一遍。

📄 状态文件

用 Markdown 记下"做到哪了、什么成了、什么挂了"，下次接着跑。

🚪 门禁

测试、类型检查、构建——能自动拦住坏结果的东西。

顺序不可跳

① 先手动跑通一次 → ② 写成 Skill → ③ 包进 Loop → ④ 最后才上定时。跳步是 loop 死在生产环境的主要原因。

4 实战案例

🏢 Anthropic 核心工程师 Boris Cherny 的工作流

他自述去年 11 月卸载了 IDE，一个月没打开过，索性删了。

现在他手下几百个小 Agent 同时跑——有的扫 GitHub issue，有的读 Slack 用户反馈，有的监控 CI 失败。每个 Agent 在自己隔离的代码分支里干活：一个写代码，另一个跑测试验收。

搞不定的才进他的收件箱，等他来做判断。自 Opus 4.5 以来，他的所有代码都是 Claude Code 写的，大部分直接在手机上完成。

📊 微软 Uber 的真实账单

Uber 给约 5000 名工程师铺开 Claude Code 后，四个月烧穿了 2026 年全年 AI 预算。人均月支出 $150–250，重度用户 $500–2000。CTO 本人一次两小时会话花了 $1200。管理层描述为"脑袋要炸"的时刻。

微软在财年末要求数千名工程师从 Claude Code 迁回 GitHub Copilot，官方理由"工具链统一"，被普遍解读为真实动因是成本。

⚠️ 三大隐蔽陷阱

偷懒：50 项安全任务做了 20 项就喊搞定

自夸：给自己的成果打高分

漂移：跑了 47 轮之后，最初的"别做 X"约束悄悄消失。测试通过了，架构却偏了。

5 ⚠️ 需要清醒的提醒

杰文斯悖论

技术效率提升，反而带来总消耗的增长。Loop 让 AI 从"叫它才动"变成"它自己一直在动"。动得越多，账单越厚。经济学的铁律从不缺席。

理解力债务

Loop 批量产出的代码没人读，团队对系统的理解深度持续下降。Addy Osmani（Loop Engineering 命名者）自省：循环产出越快，你读懂的比例越低，最舒适的选择是认知投降。

概念通胀周期

每 9 个月一轮新概念（Prompt→Context→Harness→Loop）。TCP/IP 二十年，React 五年，这个是九个月。跟上还是跟不上的焦虑本身可能就是商业策略。
